Microsoft Windows(또는 간단히 Windows)는 Microsoft에서 개발, 판매 및 판매하는 그래픽 운영 체제의 메타패밀리입니다. 여러 운영 체제 제품군으로 구성되며 각 제품군은 컴퓨팅 산업의 특정 부문에 적합합니다. 활성 Windows 제품군에는 Windows NT, Windows Embedded 및 Windows Phone이 포함됩니다. 여기에는 하위 가족이 포함될 수 있습니다. Windows Embedded Compact(Windows CE) 또는 Windows Server. 없어진 Windows 제품군에는 Windows 9x가 포함됩니다. Windows 10 Mobile은 사라진 제품군인 Windows Mobile과 관련이 없는 활성 제품입니다.
마이크로소프트는 1985년 11월 20일 그래픽 사용자 인터페이스(GUI)에 대한 관심 증가에 대응하여 MS-DOS용 그래픽 운영 체제 쉘로 Windows라는 운영 환경을 도입했습니다.[4] 마이크로소프트 윈도우는 1984년 출시된 맥 OS를 제치고 90% 이상의 점유율로 세계 개인용 컴퓨터(PC) 시장을 장악하게 됐다. PC에서 윈도우는 여전히 가장 인기 있는 운영체제다. 그러나 2014년에 Microsoft는 Android 스마트폰 판매의 엄청난 성장으로 인해 전체 운영 체제 시장의 대부분을 Android[5]에 빼앗겼다는 사실을 인정했습니다. 2014년에 판매된 Windows 장치의 수는 판매된 Android 장치의 25% 미만이었습니다. 그러나 이 비교는 두 운영 체제가 전통적으로 서로 다른 플랫폼을 대상으로 했기 때문에 완전히 관련이 없을 수 있습니다.
2016년 3월[업데이트] 기준으로 PC, 태블릿, 스마트폰 및 임베디드 장치용 Windows 최신 버전은 Windows 10입니다. 서버 컴퓨터용 최신 버전은 Windows Server 2012 R2입니다. Windows의 특수 버전은 Xbox One 게임 콘솔에서 실행됩니다.[6]

가계
마케팅 역할별
Windows 개발자인 Microsoft는 컴퓨팅 산업의 특정 부문을 대상으로 하는 Windows 운영 체제 제품군을 나타내는 여러 상표를 등록했습니다. 2014년 현재 다음 Windows 제품군이 활발하게 개발되고 있습니다.

Windows NT: 서버 컴퓨터 및 워크스테이션용 운영 체제인 Windows NT 3.1과 함께 운영 체제 제품군으로 시작되었습니다. 이제 거의 동시에 출시되고 동일한 커널을 공유하는 세 가지 운영 체제 하위 제품군으로 구성됩니다.

Windows: 주류 개인용 컴퓨터용 운영 체제. 최신 버전은 Windows 10입니다. 주제에 익숙하지 않은 사람이 특정 규칙을 준수하지 않기 때문에 이 가족의 구성원을 이름으로 식별하는 것은 거의 불가능합니다. 예를 들어 Windows Vista, Windows 7, Windows 8/8.1 및 Windows RT는 이 제품군의 구성원이지만 Windows 3.1은 그렇지 않습니다. 이 제품군의 주요 경쟁자는 Apple Inc.의 OS X입니다(운영 체제 사용 점유율 § 카테고리별 시장 점유율 참조).
Windows Server: 서버 컴퓨터용 운영 체제입니다. 최신 버전은 Windows Server 2012 R2입니다. 클라이언트 형제와 달리 강력한 명명 체계를 채택했습니다. 이 제품군의 주요 경쟁자는 Linux입니다. (c.f. 운영체제별 사용 점유율 § 카테고리별 시장 점유율)
Windows PE: 베어 메탈 컴퓨터(특히 한 번에 많은 컴퓨터에), 복구 또는 문제 해결 목적으로 Windows를 설치하는 데 사용되는 라이브 운영 체제로 작동하기 위한 Windows 형제의 경량 버전입니다. 최신 버전은 Windows PE 10.0.10586.0입니다.


Windows Embedded: 처음에 Microsoft는 완전한 컴퓨터라고 하기에는 리소스가 너무 제한된 모든 장치를 위한 범용 운영 체제로 Windows CE를 개발했습니다. 그러나 결국 Windows CE는 Windows Embedded Compact로 이름이 바뀌었고 Windows Embedded Industry, Windows Embedded Professional, Windows Embedded Standard, Windows Embedded Handheld 및 Windows Embedded Automotive로 구성된 Windows Compact 상표로 접혔습니다.[7]

다음 Windows 제품군은 더 이상 개발되지 않습니다.

Windows 9x: 소비자 시장을 겨냥한 운영 체제. 성능이 좋지 않아 중단되었습니다. (PC World는 최신 버전인 Windows ME를 역사상 최악의 제품 중 하나라고 불렀습니다.)[8] Microsoft는 이제 Windows NT로 소비자 시장을 겨냥하고 있습니다.
Windows Mobile: Windows Phone의 전신으로 휴대폰 운영 체제였습니다. 첫 번째 버전은 Pocket PC 2000이었습니다. 세 번째 버전인 Windows Mobile 2003은 Windows Mobile 상표를 채택한 첫 번째 버전입니다. 마지막 버전은 Windows Mobile 6.5입니다.
Windows Phone: 스마트폰 제조업체에게만 판매되는 운영 체제입니다. 첫 번째 버전은 Windows Phone 7이었고, 그 다음은 Windows Phone 8, 마지막 버전은 Windows Phone 8.1이었습니다. 그것은 Windows 10 Mobile에 의해 계승되었습니다.

1985년에 출시된 최초의 버전인 Windows 1.0



Windows의 역사는 컴퓨터 과학자인 Chase Bishop이 전자 장치의 첫 번째 모델을 설계하고 Interface Manager 프로젝트가 시작된 1981년 9월로 거슬러 올라갑니다. 1983년 11월(Apple Lisa 이후, Macintosh 이전)에 "Windows"라는 이름으로 발표되었지만 Windows 1.0은 1985년 11월까지 출시되지 않았습니다.[9] Windows 1.0은 Apple의 운영 체제와 경쟁할 예정이었지만 그다지 인기를 얻지는 못했습니다. Windows 1.0은 완전한 운영 체제가 아닙니다. 오히려 MS-DOS를 확장합니다. Windows 1.0의 셸은 MS-DOS Executive로 알려진 프로그램입니다. 구성 요소에는 계산기, 달력, 카드 파일, 클립보드 뷰어, 시계, 제어판, 메모장, 그림판, Reversi, 터미널 및 쓰기가 포함됩니다. Windows 1.0은 겹치는 창을 허용하지 않습니다. 대신 모든 창은 타일로 마감되어 있습니다. 다른 창 위에는 모달 대화 상자만 나타날 수 있습니다.
Windows 2.0은 1987년 12월에 출시되었으며 이전 버전보다 더 많은 인기를 얻었습니다. 그것은 사용자 인터페이스 및 메모리 관리에 대한 몇 가지 개선 사항을 제공합니다.[인용 필요] Windows 2.03은 OS를 타일 창에서 겹치는 창으로 변경했습니다. 이러한 변경의 결과로 Apple Computer는 Apple의 저작권 침해를 주장하여 Microsoft를 상대로 소송을 제기했습니다.[10][11] Windows 2.0은 또한 보다 정교한 키보드 단축키를 도입했으며 확장된 메모리를 사용할 수 있었습니다.
Windows 2.1은 Windows/286과 Windows/386의 두 가지 버전으로 출시되었습니다. Windows/386은 Intel 80386의 가상 8086 모드를 사용하여 여러 DOS 프로그램을 멀티태스킹하고 페이징 메모리 모델을 사용하여 사용 가능한 확장 메모리를 사용하여 확장 메모리를 에뮬레이트합니다. Windows/286은 이름에도 불구하고 Intel 8086 및 Intel 80286 프로세서에서 실행됩니다. 리얼 모드로 실행되지만 높은 메모리 영역을 사용할 수 있습니다.[인용 필요]
전체 Windows 패키지 외에도 타사의 초기 Windows 소프트웨어와 함께 제공되어 전체 Windows 기능 세트 없이 MS-DOS에서 Windows 소프트웨어를 실행할 수 있게 해주는 런타임 전용 버전이 있었습니다.
Windows의 초기 버전은 주로 MS-DOS에서 실행되고 파일 시스템 서비스에 사용되기 때문에 그래픽 셸로 간주됩니다.[12] 그러나 초기 Windows 버전에서도 이미 많은 일반적인 운영 체제 기능을 가정했습니다. 특히 자체 실행 파일 형식과 자체 장치 드라이버(타이머, 그래픽, 프린터, 마우스, 키보드 및 사운드)를 제공합니다. MS-DOS와 달리 Windows에서는 협력적 멀티태스킹을 통해 사용자가 동시에 여러 그래픽 응용 프로그램을 실행할 수 있습니다. Windows는 사용 가능한 메모리보다 큰 응용 프로그램을 실행할 수 있는 정교한 세그먼트 기반 소프트웨어 가상 메모리 체계를 구현했습니다. 주어진 애플리케이션이 프로세서 제어를 포기했을 때 메모리에서 이동된 데이터 세그먼트.
윈도우 3.x
주요 기사: Windows 3.0 및 Windows 3.1x




1990년에 출시된 Windows 3.0


1990년에 출시된 Windows 3.0은 설계를 개선했는데, 그 이유는 주로 Windows가 다중 작업 DOS 응용 프로그램 간에 임의의 장치를 공유할 수 있도록 하는 가상 메모리와 로드 가능한 가상 장치 드라이버(VxD) 때문입니다.[인용 필요] Windows 3.0 응용 프로그램은 보호 모드에서 실행할 수 있습니다. , 소프트웨어 가상 메모리 체계에 참여할 의무 없이 몇 메가바이트의 메모리에 액세스할 수 있습니다. 세그먼트 메모리가 어느 정도 보호를 제공하는 동일한 주소 공간 내에서 실행됩니다. Windows 3.0에서는 사용자 인터페이스도 개선되었습니다. Microsoft는 C에서 어셈블리로 중요한 작업을 다시 작성했습니다. Windows 3.0은 광범위한 상업적 성공을 달성한 최초의 Microsoft Windows 버전으로, 첫 6개월 동안 2백만 장이 판매되었습니다.[13][14]
1992년 3월 1일에 일반 출시된 Windows 3.1에는 페이스리프트가 적용되었습니다. 1993년 8월 P2P 네트워킹 기능이 통합되고 버전 번호가 3.11인 특별 버전인 Windows for Workgroups가 출시되었습니다. Windows 3.1과 함께 판매되었습니다. Windows 3.1에 대한 지원은 2001년 12월 31일에 종료되었습니다.[15]
1994년에 출시된 Windows 3.2는 Windows 3.1의 중국어 버전을 업데이트한 버전입니다.[16] 업데이트는 중국어의 복잡한 쓰기 시스템과 관련된 문제만 수정했기 때문에 이 언어 버전으로 제한되었습니다.[17] Windows 3.2는 일반적으로 기본 출력에 중국어 간체 문자가 있고 일부 번역된 유틸리티가 포함된 10디스크 버전의 MS-DOS와 함께 컴퓨터 제조업체에서 판매되었습니다.
윈도우 9x
이 부분의 본문은 Windows 9x입니다.



Windows의 다음 주요 소비자 지향 릴리스인 Windows 95는 1995년 8월 24일에 출시되었습니다. 여전히 MS-DOS 기반으로 남아 있지만 Windows 95는 기본 32비트 응용 프로그램, 플러그 앤 플레이 하드웨어, 선점형 멀티태스킹, 긴 최대 255자의 파일 이름과 이전 버전보다 향상된 안정성을 제공했습니다. Windows 95는 또한 재설계된 개체 지향 사용자 인터페이스를 도입하여 이전 프로그램 관리자를 시작 메뉴, 작업 표시줄 및 Windows 탐색기 셸로 대체했습니다. Windows 95는 Microsoft의 주요 상업적 성공이었습니다. CNET의 Ina Fried는 "Windows 95가 2001년 마침내 시장에서 출시될 무렵에는 전 세계 컴퓨터 데스크톱의 고정 장치가 되었습니다."[18] Microsoft는 Windows 95의 4가지 OEM 서비스 릴리스(OSR)를 발표했습니다. 각각은 서비스 팩과 거의 동일했습니다. Windows 95의 첫 번째 OSR은 Microsoft의 웹 브라우저인 Internet Explorer와 함께 번들로 제공되는 Windows의 첫 번째 버전이기도 합니다.[19] Windows 95에 대한 주류 지원은 2000년 12월 31일에 종료되었으며 Windows 95에 대한 확장 지원은 2001년 12월 31일에 종료되었습니다.[20]
Windows 95는 Windows 드라이버 모델, USB 복합 장치 지원, ACPI 지원, 최대 절전 모드 및 다중 모니터 구성 지원을 도입한 1998년 6월 25일 Windows 98의 출시로 이어졌습니다. Windows 98에는 Active Desktop을 통한 Internet Explorer 4와의 통합 및 Windows Desktop Update의 다른 측면도 포함되었습니다(Windows 95에서도 사용할 수 있게 된 일련의 Explorer 셸 개선 사항). 1999년 5월 Microsoft는 Windows 98의 업데이트 버전인 Windows 98 Second Edition을 출시했습니다. Windows 98 SE는 Internet Explorer 5.0과 Windows Media Player 6.2를 다른 업그레이드에 추가했습니다. Windows 98에 대한 주류 지원은 2002년 6월 30일에 종료되었으며 Windows 98에 대한 확장 지원은 2006년 7월 11일에 종료되었습니다.[21]
2000년 9월 14일 Microsoft는 Windows의 마지막 DOS 기반 버전인 Windows ME(Millennium Edition)를 출시했습니다. Windows ME는 Windows NT 기반 Windows 2000의 시각적 인터페이스 개선 사항을 통합했으며 이전 버전보다 부팅 시간이 더 빨랐습니다(그러나 리얼 모드 DOS 환경에 액세스하는 기능을 제거하고 일부 이전 프로그램과의 호환성을 제거해야 함). 22] 확장된 멀티미디어 기능(Windows Media Player 7, Windows Movie Maker, 스캐너와 디지털 카메라에서 이미지를 검색하기 위한 Windows Image Acquisition 프레임워크 포함), 시스템 파일 보호 및 시스템 복원과 같은 추가 시스템 유틸리티, 업데이트된 홈 네트워킹 도구. [23] 그러나 Windows ME는 하드웨어 호환성 문제 및 리얼 모드 DOS 지원 제거와 함께 속도와 불안정성에 대한 비판에 직면했습니다. PC World는 Windows ME를 Microsoft가 출시한 최악의 운영 체제 중 하나이자 역대 최악의 기술 제품 중 4번째로 평가했습니다.[8]
윈도우 NT
이 부분의 본문은 Windows NT입니다.
초기 버전
1988년 11월, Microsoft 내의 새로운 개발 팀(이전 Digital Equipment Corporation 개발자 Dave Cutler 및 Mark Lucovsky 포함)은 "NT OS/2"로 알려진 IBM 및 Microsoft OS/2 운영 체제의 개정된 버전에 대한 작업을 시작했습니다. NT OS/2는 POSIX와 호환되는 안전한 다중 사용자 운영 체제와 선점형 멀티태스킹 및 다중 프로세서 아키텍처를 지원하는 모듈식 이식 가능한 커널이 되도록 의도되었습니다. 그러나 Windows 3.0이 성공적으로 출시된 후 NT 개발 팀은 OS/2 대신 Win32로 알려진 Windows API의 확장된 32비트 포트를 사용하도록 프로젝트를 재작업하기로 결정했습니다. Win32는 Windows API와 유사한 구조를 유지했지만(기존 Windows 응용 프로그램을 플랫폼으로 쉽게 이식할 수 있음) 기존 NT 커널의 기능도 지원했습니다. Microsoft 직원의 승인에 따라 Windows의 첫 번째 32비트 버전인 현재 Windows NT에 대한 개발이 계속되었습니다. 그러나 IBM은 이러한 변경에 반대했고 궁극적으로 자체적으로 OS/2 개발을 계속했습니다.[24][25]
결과 운영 체제의 첫 번째 릴리스인 Windows NT 3.1(Windows 3.1과 관련하여 명명됨)은 데스크톱 워크스테이션 및 서버용 버전과 함께 1993년 7월에 출시되었습니다. Windows NT 3.5는 성능 향상과 Novell의 NetWare 지원에 중점을 두고 1994년 9월에 출시되었으며 1995년 5월에는 PowerPC 아키텍처에 대한 추가 개선 및 지원이 포함된 Windows NT 3.51이 출시되었습니다. Windows NT 4.0은 1996년 6월에 출시되어 Windows 95의 재설계된 인터페이스를 NT 시리즈에 도입했습니다. 2000년 2월 17일 Microsoft는 NT 4.0의 후속 제품인 Windows 2000을 출시했습니다. Windows 브랜드에 더 집중하기 위해 이 시점에서 Windows NT 이름은 삭제되었습니다.[25]
Windows NT의 홈 버전
Windows NT의 차기 메이저 버전인 Windows XP는 2001년 10월 25일에 출시되었습니다. Windows XP의 도입은 소비자 중심의 Windows 9x 시리즈와 아키텍처를 통합하는 것을 목표로 했습니다



이 부분의 본문은 Windows 7입니다.
2009년 7월 22일 Windows 7 및 Windows Server 2008 R2가 RTM(Release to Manufacturing)으로 출시되었으며 전자는 3개월 후인 2009년 10월 22일에 일반에 출시되었습니다. Windows 7은 Windows Vista가 이미 호환되는 응용 프로그램 및 하드웨어와 호환되는 것을 목표로 Windows 제품군에 대한 보다 집중적인 점진적 업그레이드를 의도했습니다.[31] Windows 7은 멀티 터치 지원, 업데이트된 작업 표시줄이 있는 재설계된 Windows 셸, HomeGroup[32]이라는 홈 네트워킹 시스템 및 성능 향상을 제공합니다.
윈도우 8 및 8.1
이 부분의 본문은 Windows 8입니다.
Windows 7의 후속 제품인 Windows 8은 2012년 10월 26일에 일반적으로 출시되었습니다. Windows 8에는 다음과 같은 터치 기반 장치에 최적화된 Microsoft Metro 디자인 언어를 기반으로 하는 사용자 인터페이스 도입을 포함하여 많은 중요한 변경 사항이 있었습니다. 태블릿 및 올인원 PC로. 이러한 변경 사항에는 터치 상호 작용에 더 편리하고 지속적으로 업데이트되는 정보를 표시할 수 있는 큰 타일을 사용하는 시작 화면과 주로 터치 기반 장치에서 사용하도록 설계된 새로운 종류의 앱이 포함됩니다. 기타 변경 사항에는 클라우드 서비스 및 기타 온라인 플랫폼(예: 소셜 네트워크 및 Microsoft의 자체 OneDrive(이전 SkyDrive) 및 Xbox Live 서비스)과의 통합 증가, 소프트웨어 배포를 위한 Windows Store 서비스, ARM 아키텍처를 활용하는 장치.[33][34][35][36][37][38] Windows 8.1이라고 하는 Windows 8 업데이트[39]는 2013년 10월 17일에 릴리스되었으며 새로운 라이브 타일 크기, 더 깊은 OneDrive 통합 및 기타 여러 개정판과 같은 기능을 포함합니다. Windows 8 및 Windows 8.1은 시작 메뉴 제거와 같은 일부 비판을 받았습니다.
윈도우 10
이 부분의 본문은 Windows 10입니다.
2014년 9월 30일 Microsoft는 Windows 8.1의 후속 제품으로 Windows 10을 발표했습니다. 2015년 7월 29일에 출시되었으며 Windows 8과 함께 처음 도입된 사용자 인터페이스의 단점을 해결합니다. 변경 사항에는 가상 데스크톱 시스템인 시작 메뉴의 반환, 데스크톱 대신 Windows 내에서 Windows 스토어 앱을 실행할 수 있는 기능이 포함됩니다. 전체 화면 모드보다 Windows 10은 Windows 10 응용 프로그램 가져오기(Windows 7, Windows 8.1용) 또는 Windows 업데이트(Windows 7)에서 적격 Windows 7 SP1 및 Windows 8.1 컴퓨터에서 업데이트할 수 있다고 합니다.[40]
2015년 11월 12일 Windows 10 버전 1511에 대한 업데이트가 릴리스되었습니다.[41] 이 업데이트는 Windows 7, 8 또는 8.1 제품 키와 Windows 10 제품 키를 사용하여 활성화할 수 있습니다.[42] 기능에는 새로운 아이콘과 마우스 오른쪽 버튼 클릭 컨텍스트 메뉴, 기본 프린터 관리, 시작 메뉴에서 허용되는 타일 수의 4배, 내 장치 찾기 및 Edge 업데이트가 포함됩니다.[42]
다국어 지원
다국어 지원은 Windows에 내장되어 있습니다. 키보드와 인터페이스의 언어는 지역 및 언어 제어판을 통해 변경할 수 있습니다. Input Method Editor와 같이 지원되는 모든 입력 언어에 대한 구성 요소는 Windows 설치 중에 자동으로 설치됩니다(Windows XP 이하에서는 중국어와 같은 동아시아 언어 및 아랍어와 같은 오른쪽에서 왼쪽 스크립트에 대한 파일이 필요할 수 있습니다. 해당 제어판에서도 별도로 설치해야 함). 사용자가 제공된 IME가 자신의 필요에 충분하지 않다고 느끼는 경우 타사 IME를 설치할 수도 있습니다.
운영 체제의 인터페이스 언어는 무료로 다운로드할 수 있지만 일부 언어는 Windows의 특정 버전으로 제한됩니다. LIP(언어 인터페이스 팩)는 재배포 가능하며 Microsoft 다운로드 센터에서 다운로드하여 모든 Windows 버전(XP 이상)에 설치할 수 있습니다. LIP는 Windows 인터페이스의 전부는 아니지만 대부분을 번역하고 특정 기본 언어( Windows가 원래 함께 배송된 언어). 이것은 신흥 시장에서 대부분의 언어에 사용됩니다. 전체 운영 체제를 번역하는 전체 언어 팩은 특정 버전의 Windows(Windows Vista 및 7의 Ultimate 및 Enterprise 버전, 단일 언어를 제외한 모든 Windows 8, 8.1 및 RT 버전)에서만 사용할 수 있습니다. 특정 기본 언어가 필요하지 않으며 일반적으로 프랑스어 또는 중국어와 같이 더 많이 사용되는 언어에 사용됩니다. 이러한 언어는 다운로드 센터를 통해 다운로드할 수 없지만 Windows 업데이트 서비스(Windows 8 제외)를 통해 선택적 업데이트로 사용할 수 있습니다.
설치된 응용 프로그램의 인터페이스 언어는 Windows 인터페이스 언어의 변경에 영향을 받지 않습니다. 언어의 가용성은 응용 프로그램 개발자 자신에 따라 다릅니다.
Windows 8 및 Windows Server 2012에는 인터페이스와 입력 언어를 동시에 변경할 수 있고 유형에 관계없이 언어 팩을 중앙 위치에서 다운로드할 수 있는 새로운 언어 제어판이 도입되었습니다. Windows 8.1 및 Window의 PC 설정 앱


최신 최신 버전의 Windows CE, Windows Embedded Compact 7, 개념 미디어 플레이어 UI를 표시합니다.


Windows CE(공식적으로 Windows Embedded Compact로 알려짐)는 위성 항법 시스템 및 일부 휴대폰과 같은 최소한의 컴퓨터에서 실행되는 Windows 버전입니다. Windows Embedded Compact는 Windows CE 커널이라고 하는 자체 전용 커널을 기반으로 합니다. Microsoft는 OEM 및 장치 제조업체에 Windows CE 라이선스를 제공합니다. OEM 및 장치 제조업체는 고유한 사용자 인터페이스와 경험을 수정하고 생성할 수 있으며 Windows CE는 이를 위한 기술적 기반을 제공합니다.
Windows CE는 Sega의 콘솔용 독점 OS와 함께 Dreamcast에서 사용되었습니다. Windows CE는 Windows Mobile이 파생된 핵심이었습니다. 후속 제품인 Windows Phone 7은 Windows CE 6.0 R3 및 Windows CE 7.0의 구성 요소를 기반으로 했습니다. 그러나 Windows Phone 8은 Windows 8과 동일한 NT 커널을 기반으로 합니다.
Windows Embedded Compact는 Windows NT 커널을 기반으로 하는 Windows의 모듈식 에디션인 Windows XP Embedded 또는 Windows NT 4.0 Embedded와 혼동되어서는 안 됩니다.
엑스박스 OS
이 부분의 본문은 Xbox One 시스템 소프트웨어 및 Xbox 360 시스템 소프트웨어입니다.
Xbox OS는 Xbox One에서 실행되는 Windows 버전에 부여된 비공식 이름입니다.[43] 코어 운영 체제, 게임용으로 구현된 두 번째 운영 체제 및 응용 프로그램에 대해 Windows와 유사한 환경으로 구성된 세 가지 운영 체제가 동시에 실행되므로 가상화(Hyper-V 사용)에 중점을 둔 보다 구체적인 구현입니다.[ 44] Microsoft는 매월 Xbox One의 OS를 업데이트하며 이러한 업데이트는 Xbox Live 서비스에서 Xbox로 다운로드한 후 설치하거나 PC를 통해 다운로드한 오프라인 복구 이미지를 사용하여 설치할 수 있습니다.[45] Windows 10 기반 Core는 이 업데이트에서 Windows 8 기반 Core를 대체했으며 새 시스템은 "Xbox One의 Windows 10" 또는 "OneCore"라고도 합니다.[46][47] Xbox One의 시스템은 Xbox 360과도 역호환이 가능하며[48] Xbox 360의 시스템은 원래 Xbox와 역호환됩니다.[49]

보안
Windows의 소비자 버전은 원래 네트워크 연결 없이 단일 사용자 PC에서 사용하기 쉽도록 설계되었으며 처음부터 보안 기능이 내장되어 있지 않았습니다.[60] 그러나 Windows NT 및 그 후속 제품은 보안(네트워크 포함) 및 다중 사용자 PC용으로 설계되었지만 1990년대 초반에 처음 개발되었을 때부터 인터넷 사용이 덜 널리 퍼졌습니다.[61]
프로그래밍 오류(예: 버퍼 오버플로) 및 Windows의 인기와 결합된 이러한 설계 문제는 Windows가 컴퓨터 웜 및 바이러스 작성자의 빈번한 표적이 된다는 것을 의미합니다. 2005년 6월, Bruce Schneier의 Counterpane Internet Security는 지난 6개월 동안 1,000개 이상의 새로운 바이러스와 웜을 발견했다고 보고했습니다.[62] 2005년 Kaspersky Lab은 Windows용으로 작성된 바이러스, 트로이 목마, 백도어, 익스플로잇 등 약 11,000개의 악성 프로그램을 발견했습니다.[63]
Microsoft는 Windows Update 서비스를 통해 보안 패치를 약 한 달에 한 번(보통 매월 두 번째 화요일) 출시하지만, 중요 업데이트는 필요한 경우 더 짧은 간격으로 제공됩니다.[64] Windows 2000 SP3 및 Windows XP 이후의 Windows 버전에서는 사용자가 선택하는 경우 업데이트를 자동으로 다운로드하고 설치할 수 있습니다. 결과적으로 Windows XP용 서비스 팩 2와 Windows Server 2003용 서비스 팩 1은 다른 방법보다 더 빨리 사용자에 의해 설치되었습니다.[65]
Windows 9x 시리즈는 여러 사용자에 대한 프로필을 갖는 옵션을 제공했지만 액세스 권한의 개념이 없었고 동시 액세스를 허용하지 않았습니다. 따라서 진정한 다중 사용자 운영 체제가 아닙니다. 또한 부분적인 메모리 보호만 구현했습니다. 따라서 그들은 보안 부족으로 널리 비판을 받았습니다.
대조적으로 Windows NT 시리즈 운영 체제는 진정한 다중 사용자이며 절대 메모리 보호를 구현합니다. 그러나 Windows Vista 이전에는 설치 프로세스 중에 생성된 첫 번째 사용자 계정이 새 계정의 기본값이기도 했던 관리자 계정이었기 때문에 진정한 다중 사용자 운영 체제의 많은 이점이 무효화되었습니다. . Windows XP에는 제한된 계정이 있었지만 대다수의 가정용 사용자는 더 적은 권한을 가진 계정 유형으로 변경하지 않았습니다. 부분적으로는 관리자 권한이 불필요하게 필요한 프로그램의 수 때문이었습니다. 따라서 대부분의 가정용 사용자는 항상 관리자로 실행했습니다.



Windows Vista는 사용자 계정 컨트롤이라는 권한 상승 시스템을 도입하여 이를 변경합니다[66]. 표준 사용자로 로그인하면 로그온 세션이 생성되고 가장 기본적인 권한만 포함하는 토큰이 할당됩니다. 이러한 방식으로 새 로그온 세션은 전체 시스템에 영향을 주는 변경을 수행할 수 없습니다. Administrators 그룹의 사용자로 로그인하면 두 개의 별도 토큰이 할당됩니다. 첫 번째 토큰은 일반적으로 관리자에게 부여되는 모든 권한을 포함하고 두 번째 토큰은 표준 사용자가 받는 것과 유사한 제한된 토큰입니다. 그런 다음 Windows 셸을 포함한 사용자 응용 프로그램이 제한된 토큰으로 시작되므로 관리자 계정에서도 권한이 감소됩니다. 응용 프로그램이 더 높은 권한을 요청하거나 "관리자 권한으로 실행"을 클릭하면 UAC에서 확인 메시지를 표시하고 동의하면(상승을 요청하는 계정이 관리자 그룹의 구성원이 아닌 경우 관리자 자격 증명 포함) 다음을 사용하여 프로세스를 시작합니다. 무제한 토큰.[67]
파일 권한
Windows NT 3의 모든 Windows 버전은 AGLP(Accounts, Global, Local, Permissions) AGDLP라고 하는 파일 시스템 권한 시스템을 기반으로 합니다. group'은 다른 '글로벌 그룹'을 구성원으로 포함합니다. 그런 다음 이러한 글로벌 그룹은 사용된 다른 Windows 버전에 따라 다른 그룹이나 사용자를 보유합니다. 이 시스템은 파일이나 폴더에 디렉토리에 적용되는 권한의 '정적' 할당으로 인해 Linux 및 NetWare와 같은 다른 공급업체 제품과 다릅니다. 그러나 이 AGLP/AGDLP/AGUDLP 프로세스를 사용하면 적은 수의 정적 권한을 적용할 수 있으며 파일 및 폴더에 대한 파일 권한을 다시 적용하지 않고도 계정 그룹을 쉽게 변경할 수 있습니다.
윈도우 디펜더
2005년 1월 6일 Microsoft는 이전에 출시된 Giant AntiSpyware를 기반으로 하는 Microsoft AntiSpyware 베타 버전을 출시했습니다. 2006년 2월 14일, Microsoft AntiSpyware는 Beta 2의 릴리스와 함께 Windows Defender가 되었습니다. Windows Defender는 스파이웨어 및 기타 원치 않는 소프트웨어로부터 보호하도록 설계된 프리웨어 프로그램입니다. Microsoft Windows 정품이 있는 Windows XP 및 Windows Server 2003 사용자는 Microsoft 웹 사이트에서 프로그램을 자유롭게 다운로드할 수 있으며 Windows Defender는 Windows Vista 및 7의 일부로 제공됩니다.[68] Windows 8에서는 Windows Defender와 Microsoft Security Essentials가 Windows Defender라는 단일 프로그램으로 결합되었습니다. 기능과 사용자 인터페이스를 차용하여 Microsoft Security Essentials를 기반으로 합니다. 기본적으로 활성화되어 있지만 다른 안티바이러스 솔루션을 사용하기 위해 끌 수 있습니다.[69] Windows 악성 소프트웨어 제거 도구와 Microsoft 안전 스캐너(선택 사항)는 Microsoft에서 제공하는 다른 두 가지 무료 보안 제품입니다.
제3자 분석
Symantec의 보고서를 기반으로 한 기사에서[70] Internetnews.com은 Microsoft Windows가 "2006년의 지난 6개월 동안 모니터링한 5개 운영 체제 중 가장 적은 수의 패치와 가장 짧은 평균 패치 개발 시간을 갖고 있다"고 설명했습니다. "[71]
Kevin Mitnick과 마케팅 커뮤니케이션 회사 Avantgarde가 2004년에 실시한 연구에 따르면 서비스 팩 1이 포함된 보호되지 않고 패치가 적용되지 않은 Windows XP 시스템은 손상되기 전에 인터넷에서 4분 동안 지속되었으며 보호되지 않고 패치가 적용되지 않은 Windows Server 2003 시스템은 8시간 동안 인터넷에 연결된 후 손상되었습니다.[72] Windows XP 서비스 팩 2를 실행하는 컴퓨터는 손상되지 않았습니다. 2004년 10월 AOL National Cyber ​​Security Alliance 온라인 안전 연구에 따르면 Windows 사용자의 80%가 최소한 하나의 스파이웨어/애드웨어 제품에 감염되었음을 확인했습니다.[인용 필요] Microsoft Windows 제품의 보안을 강화하는 방법을 설명하는 많은 문서가 있습니다. 일반적인 제안에는 하드웨어 또는 소프트웨어 방화벽 뒤에 Microsoft Windows 배포, 바이러스 백신 및 스파이웨어 방지 소프트웨어 실행, Windows 업데이트를 통해 제공되는 패치 설치가 포함됩니다.[73]
대체 구현
운영 체제의 인기로 인해 다른 운영 체제에 대한 호환성 계층으로 또는 Windows용으로 작성된 소프트웨어를 즉시 실행할 수 있는 독립 실행형 시스템으로 Windows 응용 프로그램과의 호환성을 제공하는 것을 목표로 하는 여러 응용 프로그램이 출시되었습니다. 여기에는 다음이 포함됩니다.

Wine – Windows API의 무료 오픈 소스 구현으로 UNIX, Linux 및 OS X를 포함한 x86 기반 플랫폼에서 많은 Windows 응용 프로그램을 실행할 수 있습니다. Wine 개발자는 이를 "호환성 계층"[74]이라고 하며 Windows 스타일 API를 사용하여 Windows 환경을 에뮬레이트합니다.


CrossOver – 라이선스 글꼴이 포함된 Wine 패키지. 개발자는 Wine에 정기적으로 기여하고 있으며 공식적으로 지원되는 응용 프로그램을 실행하는 Wine에 중점을 둡니다.
Cedega – Linux에서 Microsoft Windows 게임을 실행하기 위해 특별히 설계된 TransGaming Technologies의 독점 Wine 포크입니다. Cider로 알려진 Cedega 버전을 사용하면 Windows 게임을 OS X에서 실행할 수 있습니다. Wine은 LGPL에 따라 라이선스가 부여되었기 때문에 Cedega는 Wine에 대한 개선 사항을 독점 코드베이스로 이식할 수 없었습니다. Cedega는 2011년 2월 서비스를 중단했습니다.
Darwine – OS X 및 Darwin용 Wine 포트. QEMU에서 Wine을 실행하여 작동합니다.
Linux 통합 커널 – Linux의 일류 Windows 실행 파일(Wine DLL 사용)이 Windows 드라이버를 사용하고 해석된 Wine보다 빠를 수 있도록 하는 Linux 커널에 대한 패치 세트입니다.


ReactOS – 원래 Windows NT 4.0을 시뮬레이션하도록 설계된 Windows와 동일한 소프트웨어를 실행하기 위한 오픈 소스 OS이며 현재 Windows 7 호환성을 목표로 합니다. 1996년부터 개발 단계에 있습니다.
Linspire – 이전에 LindowsOS로, 초기에 주요 Windows 소프트웨어 실행을 목표로 만든 상업용 Linux 배포판입니다. Microsoft v. Lindows 이후에 Linspire로 이름을 변경했습니다. Xandros Desktop을 위해 단종되었습니다.
Freedows OS – GNU General Public License에 따라 출시될 x86 플랫폼용 Windows 클론을 생성하려는 오픈 소스 시도입니다. Reece K. Sellin이 1996년에 시작한 이 프로젝트는 2002년에 중단될 때까지 여러 참신한 개념을 특징으로 하는 디자인 논의 단계에 이르렀으며 완료되지 않았습니다.[75][76][77]
