Microsoft Windows (または単に Windows) は、Microsoft によって開発、販売、販売されているグラフィカル オペレーティング システムのメタファミリです。これは、オペレーティング システムのいくつかのファミリーで構成されており、それぞれがコンピューティング業界の特定のセクターに対応しています。アクティブな Windows ファミリには、Windows NT、Windows Embedded、および Windows Phone が含まれます。これらにはサブファミリーが含まれる場合があります。 Windows Embedded Compact (Windows CE) または Windows Server。廃止された Windows ファミリには、Windows 9x が含まれます。 Windows 10 Mobile はアクティブな製品であり、廃止されたファミリーの Windows Mobile とは無関係です。
マイクロソフトは、1985 年 11 月 20 日に、グラフィカル ユーザー インターフェイス (GUI) への関心の高まりに応えて、MS-DOS のグラフィカル オペレーティング システム シェルとして、Windows という名前のオペレーティング環境を導入しました。[4] Microsoft Windows は、1984 年に導入された Mac OS を追い抜き、90% 以上の市場シェアで世界のパーソナル コンピューター (PC) 市場を支配するようになりました。PC では、Windows は依然として最も人気のあるオペレーティング システムです。しかし、2014 年に Microsoft は、Android スマートフォンの売上が大幅に伸びたため、オペレーティング システム市場全体の大半を Android に奪われたことを認めました[5]。 2014 年に販売された Windows デバイスの数は、販売された Android デバイスの 25% 未満でした。ただし、この 2 つのオペレーティング システムは従来、異なるプラットフォームを対象としていたため、この比較は完全に適切ではない可能性があります。
2016 年 3 月 [更新] の時点で、PC、タブレット、スマートフォン、および組み込みデバイス用の Windows の最新バージョンは Windows 10 です。サーバー コンピューター用の最新バージョンは Windows Server 2012 R2 です。 Xbox One ゲーム コンソールでは、特殊なバージョンの Windows が実行されます。[6]

マーケティングの役割別
Windows の開発者である Microsoft は、コンピューティング業界の特定の分野を対象とする Windows オペレーティング システムのファミリを表す複数の商標を登録しています。 2014 年の時点で、次の Windows ファミリが積極的に開発されています。

Windows NT: サーバー コンピュータおよびワークステーション用のオペレーティング システムである Windows NT 3.1 と共に、オペレーティング システムのファミリとして開始されました。現在、ほぼ同時にリリースされ、同じカーネルを共有する 3 つのオペレーティング システム サブファミリで構成されています。

Windows: 主流のパーソナル コンピュータのオペレーティング システム。最新バージョンは Windows 10 です。特定のルールに準拠していないため、このファミリのメンバーを名前で特定することは、このテーマに不慣れな人にとってはほとんど不可能です。例えばWindows Vista、Windows 7、Windows 8/8.1、および Windows RT はこのファミリのメンバーですが、Windows 3.1 はそうではありません。このファミリの主な競合相手は、Apple Inc. の OS X です (オペレーティング システムの使用率 § カテゴリ別の市場シェアを参照)。
Windows Server: サーバー コンピューターのオペレーティング システム。最新バージョンは Windows Server 2012 R2 です。クライアントの兄弟とは異なり、強力な命名スキームを採用しています。このファミリの主な競合相手は Linux です。 (cf. オペレーティング システムの使用率 § カテゴリ別の市場シェア)
Windows PE: ライブ オペレーティング システムとして動作することを意図した Windows 兄弟の軽量バージョンで、Windows をベアメタル コンピューター (特に一度に多数のコンピューターに) にインストールしたり、回復またはトラブルシューティングの目的で使用したりします。最新バージョンは Windows PE 10.0.10586.0 です。


Windows Embedded: 当初、Microsoft は、本格的なコンピューターと呼ぶにはリソースが限られているすべてのデバイス用の汎用オペレーティング システムとして Windows CE を開発しました。しかし、最終的に Windows CE は Windows Embedded Compact に改名され、Windows Embedded Industry、Windows Embedded Professional、Windows Embedded Standard、Windows Embedded Handheld、および Windows Embedded Automotive から構成される Windows Compact の商標の下に折り畳まれました[7]。

次の Windows ファミリは開発されていません。

Windows 9x: 消費者市場を対象としたオペレーティング システム。最適なパフォーマンスが得られないため、廃止されました。 (PC World は、その最後のバージョンである Windows ME を史上最悪の製品の 1 つと呼びました。)[8] Microsoft は現在、Windows NT で消費者市場に対応しています。
Windows Mobile: Windows Phone の前身であり、携帯電話のオペレーティング システムでした。最初のバージョンは Pocket PC 2000 と呼ばれていました。 3 番目のバージョンである Windows Mobile 2003 は、Windows Mobile の商標を採用した最初のバージョンです。最後のバージョンは Windows Mobile 6.5 です。
Windows Phone: スマートフォンのメーカーのみに販売されるオペレーティング システム。最初のバージョンは Windows Phone 7 で、その後に Windows Phone 8 が続き、最後のバージョンは Windows Phone 8.1 でした。 Windows 10 Mobileに引き継がれました

Windows という用語は、いくつかの世代の Microsoft オペレーティング システム製品の一部またはすべてを総称しています。これらの製品は、一般的に次のように分類されます。
初期のバージョン
主な記事: Windows 1.0、Windows 2.0、および Windows 2.1x




1985年にリリースされた最初のバージョンであるWindows 1.0


Windows の歴史は 1981 年 9 月にさかのぼります。このとき、コンピューター科学者の Chase Bishop が電子デバイスの最初のモデルを設計し、Interface Manager プロジェクトが開始されました。 1983 年 11 月 (Apple Lisa の後、Macintosh の前) に「Windows」という名前で発表されましたが、Windows 1.0 は 1985 年 11 月までリリースされませんでした。 Windows 1.0 は Apple のオペレーティング システムと競合することになりましたが、あまり人気がありませんでした。 Windows 1.0 は完全なオペレーティング システムではありません。むしろ、MS-DOS を拡張します。 Windows 1.0 のシェルは、MS-DOS Executive と呼ばれるプログラムです。コンポーネントには、電卓、カレンダー、カードファイル、クリップボード ビューアー、時計、コントロール パネル、メモ帳、ペイント、リバーシ、ターミナル、および書き込みが含まれていました。 Windows 1.0 では、ウィンドウを重ねることはできません。代わりに、すべてのウィンドウが並べて表示されます。モーダル ダイアログ ボックスのみが他のウィンドウの上に表示される場合があります。
Windows 2.0 は 1987 年 12 月にリリースされ、その前身よりも人気がありました。ユーザー インターフェイスとメモリ管理にいくつかの改善が加えられています。[要出典] Windows 2.03 では、OS がタイル ウィンドウからオーバーラップ ウィンドウに変更されました。この変更の結果、Apple Computer は、Apple の著作権を侵害していると主張して、Microsoft に対して訴訟を起こすことになりました[10][11]。 Windows 2.0 では、より洗練されたキーボード ショートカットも導入され、拡張メモリを利用できるようになりました。
Windows 2.1 は、Windows/286 と Windows/386 の 2 つの異なるバージョンでリリースされました。 Windows/386 は、Intel 80386 の仮想 8086 モードを使用して複数の DOS プログラムをマルチタスクし、ページ メモリ モデルを使用して、利用可能な拡張メモリを使用して拡張メモリをエミュレートします。 Windows/286 は、その名前にもかかわらず、Intel 8086 および Intel 80286 プロセッサの両方で動作します。リアル モードで実行されますが、ハイ メモリ領域を利用できます。[要出典]
完全な Windows パッケージに加えて、サード パーティ製の初期の Windows ソフトウェアに同梱され、完全な Windows 機能セットなしで MS-DOS 上で Windows ソフトウェアを実行できるようにする、ランタイムのみのバージョンがありました。
Windows の初期のバージョンは、多くの場合、MS-DOS 上で実行され、ファイル システム サービスに使用されるため、グラフィカル シェルと見なされます。[12]ただし、最も初期の Windows バージョンでさえ、多くの一般的なオペレーティング システム機能を既に想定していました。特に、独自の実行可能ファイル形式を持ち、独自のデバイス ドライバー (タイマー、グラフィックス、プリンター、マウス、キーボード、およびサウンド) を提供します。 MS-DOS とは異なり、Windows では、協調的なマルチタスクによって、ユーザーが複数のグラフィカル アプリケーションを同時に実行することができました。 Windows は、精巧なセグメント ベースのソフトウェア仮想メモリ スキームを実装しました。これにより、利用可能なメモリよりも大きなアプリケーションを実行できます。コード セグメントとリソースはスワップインされ、メモリが不足すると破棄されます。特定のアプリケーションがプロセッサ制御を放棄したときに、データセグメントがメモリ内で移動しました。
Windows 3.x
主な記事: Windows 3.0 および Windows 3.1x




1990 年にリリースされた Windows 3.0


1990 年にリリースされた Windows 3.0 では、Windows がマルチタスク DOS アプリケーション間で任意のデバイスを共有できるようにする仮想メモリとロード可能な仮想デバイス ドライバー (VxD) により、設計が改善されました。[要出典] Windows 3.0 アプリケーションは保護モードで実行できます。 、これにより、ソフトウェア仮想メモリスキームに参加する義務なしに、数メガバイトのメモリにアクセスできます。それらは、セグメント化されたメモリがある程度の保護を提供する同じアドレス空間内で実行されます。 Windows 3.0 では、ユーザー インターフェイスも改善されました。 Microsoft は重要な操作を C からアセンブリに書き直しました。 Windows 3.0 は、最初の 6 か月で 200 万部を売り上げ、幅広い商業的成功を収めた最初の Microsoft Windows バージョンです。[13][14]
1992 年 3 月 1 日に一般公開された Windows 3.1 では、改良が施されました。 1993 年 8 月、Windows for Workgroups がリリースされました。これは、ピア ツー ピア ネットワーク機能が統合され、バージョン番号が 3.11 の特別なバージョンです。 Windows 3.1 に合わせて販売されました。 Windows 3.1 のサポートは 2001 年 12 月 31 日に終了しました。[15]
1994 年にリリースされた Windows 3.2 は、中国語版 Windows 3.1 の更新版です。[16]中国語の複雑な書記体系に関連する問題のみが修正されたため、更新はこの言語バージョンに限定されていました。 Windows 3.2 は通常、基本出力に簡体字中国語の文字が含まれ、いくつかの翻訳されたユーティリティを含む 10 ディスク バージョンの MS-DOS と共にコンピュータ メーカーによって販売されていました。
Windows 9x
主な記事: Windows 9x



Windows の次の主要な消費者向けリリースである Windows 95 が 1995 年 8 月 24 日にリリースされました。最大 255 文字のファイル名をサポートし、以前のバージョンよりも安定性が向上しました。 Windows 95 では、再設計されたオブジェクト指向のユーザー インターフェイスも導入され、以前のプログラム マネージャーが [スタート] メニュー、タスクバー、および Windows エクスプローラー シェルに置き換えられました。 Windows 95 は、Microsoft にとって商業的に大きな成功を収めました。 CNET の Ina Fried は次のように述べています。それぞれがサービス パックとほぼ同等でした。 Windows 95 の最初の OSR は、Microsoft の Web ブラウザである Internet Explorer にバンドルされた Windows の最初のバージョンでもありました。[19] Windows 95 のメインストリーム サポートは 2000 年 12 月 31 日に終了し、Windows 95 の延長サポートは 2001 年 12 月 31 日に終了しました。
Windows 95 に続いて、1998 年 6 月 25 日に Windows 98 がリリースされ、Windows ドライバー モデル、USB 複合デバイスのサポート、ACPI のサポート、休止状態、マルチモニター構成のサポートが導入されました。 Windows 98 には、Active Desktop および Windows デスクトップ アップデート (Windows 95 でも利用可能になった Explorer シェルの一連の機能強化) の他の側面による Internet Explorer 4 との統合も含まれていました。 1999 年 5 月、Microsoft は Windows 98 の更新バージョンである Windows 98 Second Edition をリリースしました。 Windows 98 のメインストリーム サポートは 2002 年 6 月 30 日に終了し、Windows 98 の延長サポートは 2006 年 7 月 11 日に終了しました。
2000 年 9 月 14 日、Microsoft は Windows の最後の DOS ベース バージョンである Windows ME (Millennium Edition) をリリースしました。 Windows ME は、Windows NT ベースの対応する Windows 2000 からビジュアル インターフェイスの拡張機能を組み込み、以前のバージョンよりも起動時間が短縮されました (ただし、リアル モードの DOS 環境にアクセスする機能を削除する必要があり、一部の古いプログラムとの互換性を削除する必要がありました)。[ 22] 拡張されたマルチメディア機能 (Windows Media Player 7、Windows ムービー メーカー、およびスキャナーやデジタル カメラから画像を取得するための Windows Image Acquisition フレームワークを含む)、システム ファイル保護やシステム復元などの追加のシステム ユーティリティ、および更新されたホーム ネットワーキング ツール。 [23]しかし、Windows ME は、ハードウェアの互換性の問題やリアル モード DOS サポートの削除に加えて、その速度と不安定性に対する批判に直面していました。 PC World は、Windows ME を Microsoft がこれまでにリリースした最悪のオペレーティング システムの 1 つであり、史上 4 番目に悪い技術製品であると見なしていました[8]。
WindowsNT
主な記事: Windows NT
初期のバージョン
1988 年 11 月、Microsoft 内の新しい開発チーム (以前の Digital Equipment Corporation の開発者である Dave Cutler と Mark Lucovsky を含む) は、「NT OS/2」として知られる IBM と Microsoft の OS/2 オペレーティング システムの改良版の作業を開始しました。 NT OS/2 は、POSIX 互換性を備えた安全なマルチユーザー オペレーティング システムと、プリエンプティブ マルチタスキングと複数のプロセッサ アーキテクチャのサポートを備えたモジュール式の移植可能なカーネルを意図していました。しかし、Windows 3.0 のリリースが成功した後、NT 開発チームは、OS/2 の代わりに Win32 として知られる Windows API の拡張 32 ビット ポートを使用するようにプロジェクトを作り直すことにしました。 Win32 は、Windows API と同様の構造を維持していました (既存の Windows アプリケーションをプラットフォームに簡単に移植できるようにする) だけでなく、既存の NT カーネルの機能もサポートしていました。 Microsoft のスタッフによる承認に続いて、現在の Windows NT (Windows の最初の 32 ビット バージョン) の開発が続けられました。しかし、IBM は変更に反対し、最終的に独自に OS/2 の開発を続けました[24][25]。
結果として生まれたオペレーティング システムの最初のリリースである Windows NT 3.1 (Windows 3.1 に関連付けて命名された) は、1993 年 7 月にデスクトップ ワークステーションおよびサーバー用のバージョンとともにリリースされました。 Windows NT 3.5 は 1994 年 9 月にリリースされ、パフォーマンスの改善と Novell の NetWare のサポートに重点が置かれ、1995 年 5 月には Windows NT 3.51 がリリースされました。これには、追加の改善と PowerPC アーキテクチャのサポートが含まれています。 Windows NT 4.0 は 1996 年 6 月にリリースされ、Windows 95 の再設計されたインターフェイスを NT シリーズに導入しました。 2000 年 2 月 17 日、Microsoft は NT 4.0 の後継である Windows 2000 をリリースしました。この時点で、Windows ブランドにより重点を置くために、Windows NT の名前は削除されました。[25]

Windows NT の次のメジャー バージョンである Windows XP は、2001 年 10 月 25 日にリリースされました。


主な記事: Windows 7
2009 年 7 月 22 日に、Windows 7 と Windows Server 2008 R2 が RTM (製造へのリリース) としてリリースされ、前者は 3 か月後の 2009 年 10 月 22 日に一般公開されました。 Windows 7 は、Windows Vista が既に対応していたアプリケーションやハードウェアとの互換性を確保することを目的として、Windows ラインへのより焦点を絞った漸進的なアップグレードを意図していました.[31] Windows 7 には、マルチタッチ サポート、更新されたタスクバーを備えた再設計された Windows シェル、HomeGroup と呼ばれるホーム ネットワーキング システム[32]、およびパフォーマンスの向上があります。
Windows 8 および 8.1
主な記事: Windows 8
Windows 7 の後継である Windows 8 は、2012 年 10 月 26 日に一般リリースされました。Windows 8 では、Microsoft の Metro デザイン言語に基づいたユーザー インターフェイスの導入や、タッチ ベースのデバイス向けの最適化など、多くの重要な変更が行われました。タブレットやオールインワン PC として。これらの変更には、タッチ操作により便利で継続的に更新された情報を表示できる大きなタイルを使用するスタート画面と、主にタッチ ベースのデバイスで使用するために設計された新しいクラスのアプリが含まれます。その他の変更には、クラウド サービスやその他のオンライン プラットフォーム (ソーシャル ネットワーク、Microsoft 独自の OneDrive (以前の SkyDrive) および Xbox Live サービスなど) との統合の強化、ソフトウェア配布用の Windows ストア サービス、Windows RT として知られる新しい亜種が含まれます。 [33][34][35][36][37][38] Windows 8.1 と呼ばれる Windows 8 の更新プログラム [39] が 2013 年 10 月 17 日にリリースされ、新しいライブ タイル サイズ、より深い OneDrive 統合、その他多くのリビジョンなどの機能が含まれています。 Windows 8 および Windows 8.1 は、スタート メニューの削除など、いくつかの批判を受けてきました。
ウィンドウズ10
主な記事: Windows 10
2014 年 9 月 30 日、Microsoft は Windows 8.1 の後継として Windows 10 を発表しました。 2015 年 7 月 29 日にリリースされ、Windows 8 で最初に導入されたユーザー インターフェイスの欠点に対処しています。変更点には、仮想デスクトップ システムであるスタート メニューの復活と、デスクトップではなくウィンドウ内で Windows ストア アプリを実行する機能が含まれます。フルスクリーンモードよりも。 Windows 10 は、Get Windows 10 Application (Windows 7、Windows 8.1) または Windows Update (Windows 7) から認定された Windows 7 SP1 および Windows 8.1 コンピューターから更新できると言われています。[40]
2015 年 11 月 12 日に、Windows 10 バージョン 1511 への更新プログラムがリリースされました。[41]この更新プログラムは、Windows 7、8、または 8.1 のプロダクト キーと Windows 10 のプロダクト キーでアクティブ化できます。[42]機能には、新しいアイコンと右クリック コンテキスト メニュー、デフォルトのプリンター管理、[スタート] メニューで許可されている 4 倍のタイル、デバイスの検索、および Edge の更新が含まれます。[42]
多言語サポート
多言語サポートは、Windows に組み込まれています。キーボードとインターフェイスの両方の言語は、[地域と言語] コントロール パネルから変更できます。 Input Method Editors など、サポートされているすべての入力言語のコンポーネントは、Windows のインストール中に自動的にインストールされます (Windows XP 以前では、中国語などの東アジア言語のファイルや、アラビア語などの右から左へのスクリプトが必要になる場合があります)。上記のコントロール パネルからも個別にインストールする必要があります)。ユーザーが提供されたものではニーズに不十分であると感じた場合は、サードパーティの IME をインストールすることもできます。
オペレーティング システムのインターフェイス言語は無料でダウンロードできますが、一部の言語は Windows の特定のエディションに限定されています。言語インターフェイス パック (LIP) は再配布可能で、Microsoft のダウンロード センターからダウンロードして、Windows の任意のエディション (XP 以降) にインストールできます。すべてではありませんがほとんどの Windows インターフェイスを翻訳し、特定のベース言語 ( Windows が最初に出荷した言語)。これは、新興市場のほとんどの言語で使用されています。完全なオペレーティング システムを翻訳する完全な言語パックは、Windows の特定のエディション (Windows Vista および 7 の Ultimate および Enterprise エディション、単一言語を除く Windows 8、8.1、および RT のすべてのエディション) でのみ使用できます。特定のベース言語を必要とせず、フランス語や中国語などのより一般的な言語で一般的に使用されます。これらの言語は、ダウンロード センターからダウンロードすることはできませんが、Windows Update サービス (Windows 8 を除く) を通じてオプションの更新プログラムとして入手できます。
インストールされているアプリケーションのインターフェイス言語は、Windows インターフェイス言語の変更の影響を受けません。言語の可用性は、アプリケーション開発者自身に依存します。
Windows 8 および Windows Server 2012 では、インターフェイスと入力言語の両方を同時に変更できる新しい言語コントロール パネルが導入され、言語パックは種類に関係なく中央の場所からダウンロードできます。 Windows 8.1 および Window の PC 設定アプリ



コンセプト メディア プレーヤー UI を表示する Windows CE の最新バージョンである Windows Embedded Compact 7。


Windows CE (正式には Windows Embedded Compact として知られています) は、衛星航法システムや一部の携帯電話などの最小限のコンピューターで動作する Windows のエディションです。 Windows Embedded Compact は、Windows CE カーネルと呼ばれる独自の専用カーネルに基づいています。 Microsoft は Windows CE を OEM およびデバイス メーカーにライセンス供与します。 OEM とデバイス メーカーは独自のユーザー インターフェイスとエクスペリエンスを変更および作成できますが、Windows CE はそのための技術基盤を提供します。
Windows CE は、コンソール用のセガ独自の独自の OS と共にドリームキャストで使用されました。 Windows CE は、Windows Mobile が派生したコアでした。その後継の Windows Phone 7 は、Windows CE 6.0 R3 と Windows CE 7.0 の両方のコンポーネントに基づいていました。ただし、Windows Phone 8 は、Windows 8 と同じ NT カーネルに基づいています。
Windows Embedded Compact を、Windows XP Embedded または Windows NT 4.0 Embedded、Windows NT カーネルに基づく Windows のモジュラー エディションと混同しないでください。
Xbox OS
主な記事: Xbox One システム ソフトウェアおよび Xbox 360 システム ソフトウェア
Xbox OS は、Xbox One で動作する Windows のバージョンに付けられた非公式の名前です。[43]仮想化 (Hyper-V を使用) に重点を置いたより具体的な実装であり、コア オペレーティング システム、ゲーム用に実装された 2 番目のオペレーティング システム、およびアプリケーション用のより Windows に似た環境で構成される 3 つのオペレーティング システムが同時に実行されるためです。[ 44] Microsoft は Xbox One の OS を毎月更新します。これらの更新は、Xbox Live サービスから Xbox にダウンロードしてインストールするか、PC 経由でダウンロードしたオフラインの回復イメージを使用してインストールできます。[45] Windows 10 ベースのコアは、この更新プログラムで Windows 8 ベースのコアに取って代わり、新しいシステムは「Xbox One の Windows 10」または「OneCore」と呼ばれることがあります。[46][47] Xbox One のシステムは Xbox 360 との下位互換性も備えており[48]、Xbox 360 のシステムはオリジナルの Xbox と下位互換性があります[49]。

安全
Windows のコンシューマ バージョンは、もともとネットワーク接続のないシングル ユーザー PC で使いやすいように設計されており、最初からセキュリティ機能が組み込まれていませんでした[60]。ただし、Windows NT とその後継は、セキュリティ (ネットワーク上を含む) とマルチユーザー PC 向けに設計されていますが、当初はインターネット セキュリティをそれほど考慮して設計されていませんでした。 [61]
これらの設計上の問題は、プログラミング エラー (バッファ オーバーフローなど) と組み合わされており、Windows の人気は、コンピュータ ワームやウイルス作成者の標的になることが多いことを意味します。 2005 年 6 月、Bruce Schneier の Counterpane Internet Security は、過去 6 か月間に 1,000 を超える新しいウイルスとワームを確認したと報告しました。 2005 年、Kaspersky Lab は約 11,000 の悪意のあるプログラム (ウイルス、トロイの木馬、バックドア、および Windows 用に作成されたエクスプロイト) を発見しました[63]。
マイクロソフトは、Windows Update サービスを通じて、月に 1 回 (通常は第 2 火曜日) にセキュリティ パッチをリリースしますが、重要な更新プログラムは、必要に応じてより短い間隔で提供されます.[64] Windows 2000 SP3 および Windows XP 以降のバージョンの Windows では、ユーザーが選択した場合、更新プログラムを自動的にダウンロードしてインストールできます。その結果、Windows XP の Service Pack 2 と Windows Server 2003 の Service Pack 1 は、他の方法よりも迅速にインストールされました。[65]
Windows 9x シリーズでは、複数のユーザーのプロファイルを持つオプションが提供されていましたが、アクセス特権の概念がなく、同時アクセスが許可されていませんでした。真のマルチユーザー オペレーティング システムではありませんでした。さらに、部分的なメモリ保護のみを実装しました。したがって、彼らはセキュリティの欠如について広く批判されました。
対照的に、Windows NT シリーズのオペレーティング システムは真のマルチユーザーであり、絶対的なメモリ保護を実装しています。ただし、真のマルチユーザー オペレーティング システムであることの多くの利点は、Windows Vista より前のセットアップ プロセスで作成された最初のユーザー アカウントが管理者アカウントであり、新しいアカウントの既定値でもあったという事実によって無効化されていました。 . Windows XP には限られたアカウントしかありませんでしたが、ほとんどのホーム ユーザーは権限の少ないアカウント タイプに変更しませんでした。その理由の 1 つは、不必要に管理者権限を必要とするプログラムが多かったためです。そのため、ほとんどのホーム ユーザーは常に管理者として実行されていました。


Windows Vista は、ユーザー アカウント制御と呼ばれる特権昇格システムを導入することで、これを変更します[66]。標準ユーザーとしてログインすると、ログオン セッションが作成され、最も基本的な権限のみを含むトークンが割り当てられます。このように、新しいログオン セッションは、システム全体に影響を与える変更を行うことができません。 Administrators グループのユーザーとしてログインすると、2 つの個別のトークンが割り当てられます。最初のトークンには通常管理者に与えられるすべての権限が含まれ、2 番目のトークンは標準ユーザーが受け取るものと同様の制限付きトークンです。次に、Windows シェルを含むユーザー アプリケーションが制限されたトークンで開始されるため、管理者アカウントであっても権限が制限された環境になります。アプリケーションがより高い権限を要求するか、[管理者として実行] をクリックすると、UAC は確認を求め、同意が得られた場合 (昇格を要求するアカウントが管理者グループのメンバーでない場合は管理者の資格情報を含む)、次を使用してプロセスを開始します。無制限のトークン。 [67]
ファイルのアクセス許可
Windows NT 3 以降のすべての Windows バージョンは、AGLP (アカウント、グローバル、ローカル、アクセス許可) AGDLP と呼ばれるファイル システム アクセス許可システムに基づいています。このグループには、他の「グローバル グループ」がメンバーとして含まれます。これらのグローバル グループは、使用されているさまざまな Windows バージョンに応じて、他のグループまたはユーザーを保持します。このシステムは、ファイルまたはフォルダーに適用されるディレクトリのアクセス許可が「静的」に割り当てられるため、Linux や NetWare などの他のベンダー製品とは異なります。ただし、この AGLP/AGDLP/AGUDLP のプロセスを使用すると、少数の静的アクセス許可を適用でき、ファイルとフォルダーのファイル アクセス許可を再適用することなく、アカウント グループを簡単に変更できます。
Windows ディフェンダー
2005 年 1 月 6 日、Microsoft は、以前にリリースされた Giant AntiSpyware に基づいて、Microsoft AntiSpyware のベータ版をリリースしました。 2006 年 2 月 14 日の Beta 2 のリリースにより、Microsoft AntiSpyware は Windows Defender になりました。Windows Defender は、スパイウェアやその他の望ましくないソフトウェアから保護するために設計されたフリーウェア プログラムです。 Microsoft Windows の正規のコピーを持っている Windows XP および Windows Server 2003 のユーザーは、Microsoft の Web サイトからプログラムを自由にダウンロードでき、Windows Defender は Windows Vista および 7 の一部として出荷されます。 Windows 8 では、Windows Defender と Microsoft Security Essentials が、Windows Defender という名前の 1 つのプログラムに統合されました。 Microsoft Security Essentials に基づいており、その機能とユーザー インターフェイスを借用しています。デフォルトで有効になっていますが、別のウイルス対策ソリューションを使用するためにオフにすることができます。 [69] Windows Malicious Software Removal Tool とオプションの Microsoft Safety Scanner は、Microsoft が提供する他の 2 つの無料セキュリティ製品です。
第三者分析
Symantec のレポートに基づく記事[70]で、internetnews.com は、Microsoft Windows について、「2006 年の最後の 6 か月間に監視した 5 つのオペレーティング システムの中で、パッチの数が最も少なく、パッチの平均開発時間が最も短かった」と説明しています。 "[71]
2004 年に Kevin Mitnick とマーケティング コミュニケーション会社 Avantgarde が行った調査によると、保護もパッチも適用されていない Service Pack 1 を適用した Windows XP システムは、インターネット上でわずか 4 分間持続しただけで侵害され、保護もパッチも適用されていない Windows Server 2003 システムは、インターネットに 8 時間接続した後、侵害されました。 [72] Windows XP Service Pack 2 を実行していたコンピューターは侵害されませんでした。 2004 年 10 月の AOL National Cyber​​ Security Alliance Online Safety Study では、Windows ユーザーの 80% が少なくとも 1 つのスパイウェア/アドウェア製品に感染していると判断されました。典型的な提案には、Microsoft Windows をハードウェアまたはソフトウェア ファイアウォールの内側に配置する、ウイルス対策ソフトウェアおよびスパイウェア対策ソフトウェアを実行する、Windows Update を通じて利用可能になったときにパッチをインストールするなどがあります[73]。
代替実装
オペレーティング システムの人気により、Windows アプリケーションとの互換性を提供することを目的とした多くのアプリケーションがリリースされています。これは、別のオペレーティング システムの互換レイヤーとして、または Windows 用に書かれたソフトウェアをすぐに実行できるスタンドアロン システムとして提供されます。これらには以下が含まれます：

Wine – Windows API の無料でオープンソースの実装であり、UNIX、Linux、OS X を含む x86 ベースのプラットフォームで多くの Windows アプリケーションを実行できます。Wine 開発者はこれを「互換レイヤー」[74] と呼んでいます。 Windows スタイルの API を使用して Windows 環境をエミュレートします。


CrossOver – ライセンス付きフォントを含む Wine パッケージ。その開発者は Wine に定期的に貢献しており、公式にサポートされているアプリケーションを実行する Wine に焦点を当てています。
Cedega – TransGaming Technologies による Wine の独自のフォークで、Linux で Microsoft Windows ゲームを実行するために特別に設計されています。 Cider として知られる Cedega のバージョンを使用すると、Windows ゲームを OS X で実行できます。Wine は LGPL の下でライセンスされていたため、Cedega は Wine に加えられた改善を独自のコードベースに移植することができませんでした。 Cedega は 2011 年 2 月にサービスを終了しました。
Darwine – OS X および Darwin 用の Wine のポート。 QEMU上でWineを実行することで動作します。
Linux Unified Kernel – Linux カーネルへの一連のパッチで、Linux の一流の Windows 実行可能ファイル (Wine DLL を使用) が Windows ドライバーを使用し、解釈された Wine よりも高速になります。


ReactOS – Windows と同じソフトウェアを実行することを目的としたオープンソース OS。当初は Windows NT 4.0 をシミュレートするように設計されていましたが、現在は Windows 7 との互換性を目指しています。 1996 年から開発段階にある。
Linspire – 以前は LindowsOS で、最初は主要な Windows ソフトウェアを実行することを目的として作成された商用 Linux ディストリビューションです。 Microsoft v. Lindows の後、名前を Linspire に変更。 Xandros Desktop を支持して廃止されました。
Freedows OS – x86 プラットフォーム用の Windows クローンを作成するオープンソースの試みで、GNU General Public License の下でリリースされることを意図しています。 1996 年に Reece K. Sellin によって開始されたこのプロジェクトは、2002 年に中断されるまで、多くの斬新なコンセプトを特徴とするデザインの議論の段階にしか達せず、決して完成することはありませんでした.[75][76][77]
